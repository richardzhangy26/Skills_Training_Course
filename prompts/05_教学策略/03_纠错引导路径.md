## 评分标准

满分: 5分

### 分数段标准

| 分数 | 评级 | 说明 |
|------|------|------|
| **5分** | 优秀 | 学生答错时从不直接给答案，而是提供恰当的线索或引导性问题，帮助学生自己发现错误并找到正确答案。纠错艺术高超 |
| **4分** | 良好 | 多数时候能提供引导（70%以上），偶尔1-2次直接给出答案或纠正 |
| **3分** | 合格 | 半数时候能提供引导（40%-60%），或引导不够充分（线索过于明显接近直接给答案） |
| **2分** | 不足 | 多数时候直接给答案或纠正（60%以上），引导不足 |
| **0-1分** | 无 | 学生答错时完全直接给答案，无任何引导过程 |

### 扣分细则（原文内容）

- 统计学生答错时智能体的应对方式
- 评估引导的有效性（线索是否恰当，既不过于明显也不过于隐晦）
- 关注是否帮助学生自己发现错误

### 评分要点（原文内容）

- 统计学生答对后智能体是否进行深度追问
- 评估追问的质量（是否真正引导深入思考）
- 关注追问的多样性（"为什么""怎么理解""能举例吗""如果...会怎样"等）

你必须按照以下JSON格式输出评分结果:

```json

  "sub_dimension": "纠错引导路径",
  "score": 0, // 替换为该分项的实际得分(数字)
  "full_score": 5,
  "rating": "合格", // 替换为实际评级 (优秀/良好/合格/不足/较差)
  "score_range": "", // 替换为实际落入的分数段
  "judgment_basis": "此处填写详细的得分理由...", // 必须基于事实分析
  "issues": [
    {
      "description": "此处填写具体问题描述",
      "location": "第1轮对话",
      "quote": "此处引用对话原文",
      "severity": "medium",
      "impact": "问题影响简述"
    }
  ]
}
```

**字段说明:**
- score: 必须是数字
- rating: 必须是 "优秀"/"良好"/"合格"/"不足"/"较差" 之一
- severity: 必须是 "high"/"medium"/"low" 之一

**关键要求:**
1. **绝不要直接复制上面的示例值！** 你必须根据实际对话内容重新生成所有字段的值。
2. **先判定分数段**: 根据整体表现确定属于哪个分数段
3. **再列举问题**: 详细列出导致该分数段判定的具体问题
4. **强制证据引用**: 每个问题必须有明确的位置定位和原文引用
5. **quote字段必须是对话中的实际内容**, 不能编造
6. **location必须精确到第X轮对话**
7. **judgment_basis格式要求**:
   - 使用 **Markdown** 格式进行排版，优化阅读体验
   - 使用 **粗体** 强调关键结论
   - 使用列表整理具体分析点，避免大段纯文本
   - 条理清晰，分段落阐述
8. **特别注意**: 字符串内部的双引号必须转义 (例如使用 \" 而不是 "), 确保JSON格式合法

请严格按JSON格式输出，不要有任何多余的文字！
