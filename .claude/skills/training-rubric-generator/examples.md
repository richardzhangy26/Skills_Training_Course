### 生成的评价标准（片段）

#### 评分项：汽蚀特征识别 (3分)

**用户描述**:
考查学生是否能准确识别离心泵汽蚀现象的特征。

**大模型评判参考**:
评判学生是否能准确识别离心泵汽蚀现象的特征，包括泵发出异常声音、出口压力波动、泵效率下降等现场表现。还要评估学生是否能将这些现象与汽蚀现象关联起来。

关键评判点：
- 是否提到了异常声音特征
- 是否提到了压力波动
- 是否理解现象与汽蚀的关系

常见错误：
- 只提到声音或压力，没有系统性分析
- 混淆汽蚀与其他故障现象
- 不能准确描述现象的具体表现

评分逻辑：
- 3分：准确列举了3个以上汽蚀特征，能清晰关联现场现象与汽蚀现象
- 2分：提到了2-3个特征，基本理解汽蚀现象
- 1分：只提到1个特征或理解不够准确
- 0分：未能正确识别特征或理解错误

**优秀示例**:
回答：泵发出"咔咔"的异常声响，出口压力表指针晃动厉害，这些都是汽蚀的典型特征。汽蚀是因为进口压力太低，液体汽化了，所以会有气泡，气泡进入压缩区后坍塌，就会产生冲击声和压力波动。

说明：这个回答准确指出了3个关键特征（声音、压力波动、液体汽化），并能解释现象产生的根本原因，逻辑严密。

**不足示例1**:
回答：泵声音不对，应该是故障了。

问题：描述过于笼统，没有具体指出是什么声音，也没有提到其他特征，缺少诊断依据。

**不足示例2**:
回答：可能是密封圈坏了，所以有声音和压力不稳定。

问题：混淆了两种不同的故障现象，没有准确识别汽蚀的特征，分析方向错误。

### 生成的评价标准（片段）

#### 评分项：知识掌握 (3分)

**用户描述**:
是否掌握电力系统的基本知识和概念。

**大模型评判参考**:
评判学生是否掌握了电力系统的基本知识和概念。需要检查学生对关键知识点的理解深度，是否能准确陈述，是否理解了知识之间的逻辑关系。

关键评判点：
- 对核心概念的理解是否准确
- 是否能准确陈述关键知识点
- 是否理解了不同知识点之间的逻辑关系
- 是否能举例说明

常见错误：
- 对概念理解不准确，存在混淆
- 知识点记住了但不理解其含义
- 无法区分相似的概念
- 不能用自己的话解释

评分逻辑：
- 3分：完全满足要求，表述准确、完整、清晰
- 2分：基本满足要求，少量不足
- 1分：部分满足要求
- 0分：严重不足或完全不满足

**优秀示例**:
回答：电力系统由发电、输电、变电、配电和用电五个环节组成。发电厂将原始能量转化为电能，不同类型的发电厂工作原理不同，如火电厂通过燃烧煤炭产生蒸汽驱动汽轮机。

说明：此回答系统地阐述了电力系统的基本结构，能够准确列举核心概念，并能用具体例子说明，表明学生对知识有完整的理解。

**不足示例1**:
回答：电力系统很重要，我们要学好。

问题：没有具体的知识内容，无法判断学生是否真正掌握了核心概念。

**不足示例2**:
回答：有发电厂和变电站，发电的和改变的。

问题：虽然提到了组件，但描述不准确，"改变的"表述不专业，说明对概念理解不深。

## 常见使用场景

### 场景1：快速补充现有标准的细节

**需求**: 文档中有基础的评价标准表，需要补充详细的评判指南和示例

**解决方案**:
```bash
python rubric_generator.py task.md
```

**结果**:
- ✅ 保留原有的评分项和分值
- ✅ 自动补充大模型评判指南
- ✅ 生成正反例和评分逻辑
- ✅ 输出规范的JSON和Markdown

### 场景2：为新课程生成完整的评价标准

**需求**: 新课程还没有评价标准，需要从零开始创建

**解决方案**:
```bash
python rubric_generator.py new_task.md
```

**结果**:
- ✅ 识别课程类型
- ✅ 自动生成3-5个评分项
- ✅ 生成详细的评判指南
- ✅ 提供正反例参考

### 场景3：为大模型评判做准备

**需求**: 已经有评价标准，但需要为大模型的自动评判做充分准备

**解决方案**:
```bash
# 生成评价标准（包括详细的aiGuidelines）
python rubric_generator.py task.md

# 使用生成的JSON文件中的aiGuidelines指导大模型评判
# 大模型会参考：
# - description: 200-300字的详细要求
# - keyPoints: 关键评判点
# - commonMistakes: 常见错误提示
# - scoringLogic: 按分值等级的评分标准
```

**结果**:
- ✅ 大模型评判更准确
- ✅ 减少误判和评分波动
- ✅ 评判更透明和可解释

## 完整的工作流程示例

假设你有一个新的实训任务文档：`任务1-离心泵故障诊断.md`

```bash
# 第一步：生成基础配置和封面图
cd .claude/skills/training-config-setup
python config_generator.py /path/to/任务1-离心泵故障诊断.md

# 输出：基础配置.json, 封面图提示词.txt, 封面图(URL)

# 第二步：生成详细的评价标准（<- 你在这里）
cd ../training-rubric-generator
python rubric_generator.py /path/to/任务1-离心泵故障诊断.md

# 输出：评价标准.json, 评价标准.md

# 第三步：生成完整的训练剧本
# 使用 training-script-generator skill
# 输出：训练剧本配置.md

# 第四步：模拟对话流程
# 使用 training-dialogue-simulator skill
# 输出：对话流程模拟.md

# 最终目录结构：
# 任务1-离心泵故障诊断/
# ├── 基础配置.json
# ├── 评价标准.json
# ├── 评价标准.md
# ├── 训练剧本配置.md
```
