# homework-answer-generator

根据作业批阅文档生成5个不同档次的学生答案样本（优秀、良好、中等、及格、不及格），每份答案严格对应评分标准和档次描述，用于测试作业批阅系统的准确性。

## 使用方法

用户可以通过以下方式触发：
- "根据这个作业批阅文档生成5个档次的学生答案"
- "生成作业答案样本"
- "@作业批阅文档.docx 生成学生答案"

## 输入

- docx格式的作业批阅文档，包含：
  - 作业描述（任务要求、提交要求等）
  - 评分标准（评价维度、评价内容、分值）
  - 档次描述（优秀、良好、中等、及格、不及格的分数区间和描述）

## 输出

- 5份学生答案文件（优秀档答案.md、良好档答案.md、中等档答案.md、及格档答案.md、不及格档答案.md）
- 1份评分对照表（评分对照表.md）

## 工作流程

### 第一步：读取并转换docx文档

1. 检查用户是否提供了docx文件路径，如果没有则询问
2. 使用Bash工具调用 `./scripts/convert_docx_to_markdown.py` 将docx转换为markdown
3. 读取转换后的markdown内容

### 第二步：解析文档结构

从markdown中提取以下信息：
- **作业名称**：从文件名或文档标题提取
- **作业描述**：识别"作业描述"、"任务要求"、"作业要求"等关键词
- **评分标准**：识别"评分标准"、"评价维度"、"评分细则"等关键词，解析表格中的评价维度、评价内容、分值
- **档次描述**：识别"总体评价档次"、"评分档次"、"优秀档"、"良好档"等关键词，提取5个档次的分数区间和描述

### 第三步：为每个档次生成答案

按照优秀→良好→中等→及格→不及格的顺序依次生成：

**优秀档（90-100分）**：
- 覆盖100%评分点
- 分析深入全面，有独特见解
- 语言专业严谨，结构完整
- 无明显不足

**良好档（80-89分）**：
- 覆盖90%左右评分点
- 分析较好但某些细节深度略有欠缺
- 语言规范流畅
- 某些细节或深度稍有不足

**中等档（70-79分）**：
- 覆盖70%左右评分点
- 基本达标但缺乏创意和深度
- 语言较为口语化
- 创意性、深度欠缺

**及格档（60-69分）**：
- 覆盖60%左右评分点
- 内容浅显简单
- 表达不够准确
- 多处明显不足

**不及格档（<60分）**：
- 覆盖<50%评分点
- 内容严重缺失或存在明显错误
- 逻辑混乱或表达不清
- 关键内容缺失

### 第四步：生成评分对照表

汇总各档次的预期得分，生成评价维度得分分布表和使用说明。

### 第五步：保存输出文件

1. 在文档所在目录创建以作业名称命名的文件夹
2. 保存5份答案文件
3. 保存评分对照表

## 输出文件格式

### 单份答案文件格式

```markdown
# {档次名称}档答案


## 答案内容
{生成的学生答案内容}

---

## 生成提示词模板

### 系统提示词

```
你是一位经验丰富的教育评估专家，擅长根据评分标准生成真实可信的学生作业答案。

你的任务是生成一份{档次名称}档的学生作业答案，该档次的特点是：
{档次描述}

要求：
1. 答案必须真实可信，像真实学生写的，避免过于模板化
2. 严格对应评分标准，该档次应覆盖约{覆盖率}%的评分点
3. 语言风格符合该档次特征（优秀档专业严谨，中等档口语化等）
4. 答案长度根据作业类型自动调整
5. 避免生成免责声明、标题、署名等额外内容
```

### 用户提示词

```
## 作业描述
{作业描述}

## 评分标准
{评分标准表格}

## 要求
请生成一份{档次名称}档（{分数区间}分）的学生答案。

该档次的核心要求：
{档次核心要求}

请直接输出答案内容，不要添加任何说明、标题或署名。
```

## 工具权限

- Read: 读取转换后的markdown文件
- Write: 创建输出目录和文件
- Bash: 调用docx转换脚本

## 依赖

- `./scripts/convert_docx_to_markdown.py` 脚本必须存在于工作目录
- Python环境需安装必要的依赖（python-docx, mammoth等）
